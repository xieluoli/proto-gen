# PRD bullets 写法规范

> **适用范围**：设备无关。PC · macOS / Mobile 系列共用同一套 PRD 写法。

每个 section 的 PRD bullets 按**从上到下视觉布局顺序**排列，顶部控件 → 主区内容 → 操作按钮 → 弹窗/抽屉。

## 元素描述模板

每条 bullet **以加粗元素名作标题前缀**（`<strong>元素名</strong>：` / md 里 `**元素名**：`），冒号后接「展示...」描述：

| 元素类型 | 描述模板 |
|------|------|
| 静态文本/标题 | `**{元素名}**：展示「{文案}」{位置/样式}` |
| 内容卡片/列表 | `**{元素名}**：展示 {元素}，内容来源：{数据源}` |
| 按钮 | `**{按钮名}**：展示「{文案}」按钮（{样式}），点击后{行为}` |
| 链接 | `**{链接名}**：展示「{文案}」链接，点击后跳转至 {目标}` |
| 状态指示 | `**{状态名}**：展示 {状态元素}（{文案/颜色}），用于 {含义}` |
| 交互（hover/focus） | `**{交互名}**：{触发行为}时{反馈}` |
| 条件分支 | `**{元素名}**：{条件 A}时{元素状态描述}；{条件 B}时{元素另一状态描述}` |

规则：

- 每条单一职责，一个可感知元素一条
- **加粗标题前缀**：以 `**元素名**：`（HTML 面板 `<strong>元素名</strong>：`）起头，元素名取简短可感知的控件/区域名（如「登录按钮」「积分余额」「套餐卡片」），冒号后接「展示...」描述
- 描述含"是什么"和"什么数据/什么行为"两维
- 数据来源用通俗描述（来自配置 / 来自后端 / 静态 / 由用户输入），不写实现细节
- 冒号使用全角 `：`

### 条件分支型描述

同一控件在不同上下文有不同行为时，用**条件对照**清晰列出每种分支，避免歧义。

示例：

> - **删除图标·默认渠道**：渲染为禁用态（灰色 + not-allowed 光标 + 透明 hover 背景），tooltip 文案为「默认渠道不可删除」；点击无任何响应
> - **删除图标·非默认渠道**：使用危险色样式，用于强调破坏性操作

写法要点：

- 每个分支独立一条 bullet（不要在同一行用「如果…则…」并列描述）
- 条件前缀简短（如「默认渠道」「失败态」「未登录用户」），冒号后描述视觉/行为
- 用于同控件不同态、同元素不同用户身份、同字段在不同流程的差异

## 重复内容处理

同一模块内多个 section 共享相同结构时，只在**第一个 section 完整描述**，后续 section 改用引用，降低维护成本。

### 四类常见重复场景

| 场景 | 示例 | 引用写法 |
|------|------|---------|
| **外壳级通用模块**（顶部窗口栏 / 侧边栏 / 顶部工具栏）在同模块所有 section 重复出现 | 页面-01 / 页面-02 / 页面-03（同 app-shell 套同一套外壳） | 第一个 section 完整描述这些通用模块；**后续 section 一律省略，不写「与首页保持一致」之类废话引用**，只列主区差异；除非外壳本身有差异（如某 tab 隐藏侧栏某项）才单独写一行差异说明 |
| **详情页多状态变体**（同模板、不同主按钮/badge） | 详情页-01 / 详情页-02 / 详情页-03（不同状态） | `详情页通用结构（返回链接、图标 + 名称 + 副标题、完整简介、操作按钮）同 详情页-01` |
| **弹窗通用结构**（遮罩 + X 关闭按钮）在同模块多处弹窗 | 弹窗-01 / 弹窗-02（不同弹窗） | `遮罩、X 关闭按钮同 弹窗-01；底层为 页面-01 dimmed 态` |
| **遮罩在同文件连续出现**（抽屉/弹窗叠加） | 弹窗-02 → 弹窗-03 | `遮罩同上；[本节独有内容]` |

> **外壳级模块的特殊性**：与详情页 / 弹窗 / 遮罩三类「内容级」重复不同，外壳（顶部窗口栏 / 侧边栏 / 顶部工具栏）是**整个应用共用**的，写 1 次足够。后续 section 写「侧边栏与首页一致」这种引用句本身就是赘述——直接删，**留白即默认外壳无变化**。只有当外壳真的变了（如某侧栏列表保留上次缓存这种差异），才独立写一条 bullet 说差异。

### 引用规则

1. 第一个引入该结构的 section 必须完整描述，它是后续 section 的锚点
2. 后续 section 先写一行引用，再列差异点（用「差异：」或直接列差异 bullets）
3. 差异内容仍按正常描述模板写，不省略
4. 若两个弹窗内容差异较大，不强制引用，完整描述更清晰

### 正面案例

「编辑」/「编辑-默认」/「删除」三个 section 的引用式差异描述典范：

- 「编辑」完整描述编辑子页结构
- 「编辑-默认」首行 `结构与「编辑」一致，差异点见下`，之后只列差异（默认项额外的 badge、只读字段、恢复按钮等）
- 「删除」首行 `底层背景为「编辑」子页内容...`，复用底层描述，重点写弹窗本身

## 改动型原型：功能概览只写本次改动

当原型是给**既有页面**新增 / 修改功能的迭代切片（而非全新页面）时，功能概览**只描述本次改动引入或改变的元素**，既有功能一律不复述。

- ❌ 反例：给某详情页新增一个配置区，却把返回入口、状态徽标、主操作按钮、依赖列表、删除区等既有元素全列一遍 —— 冗余
- ✅ 正例：只写新增的配置区及其中的开关 / 入口卡片；既有头部、列表、操作区一概不写
- **判断标准**：这条 bullet 描述的元素，是本次改动**新增或改变**的吗？不是 → 删
- 全新页面原型不受此约束（所有元素都是「本次」，照常从上到下枚举）
- section 标题可点明改动主题（如「XX 页 · 新增 XX 区」），让读者一眼知道本切片只讲增量

## PRD 写作风格（人类视角，不写代码语言）

PRD 旁注的读者是 **产品 / 测试 / 研发**——三类都是人。写法要保证 3 个角色都能 0 上下文读懂：

### 1. 用产品语言，禁止代码语言

| ❌ 反例 | ✅ 正例 |
|---|---|
| `sidebar-brand 用 .brand-mark + h2.brand-name 渲染` | 左侧导航栏品牌区：顶部展示品牌 logo + 文字 |
| `AgentStatusBadge variant=success` | 卡片右上角展示绿色「已安装」徽标 |
| `onClick → router.push('/detail/:id')` | 点击后前往对应详情页 |
| `installedCount > 0 ? showBadge : null` | 当任一数字为 0 时该项徽标自动隐藏，让顶栏保持简洁 |

### 2. 系统级元素描述要平台无关

涉及操作系统、浏览器、设备等会变的元素，描述用「跟随...呈现」「按...而定」「原型图仅展示 X 风格作示意」，不要写死具体形态：

- ❌ 「窗口栏左上展示红黄绿三个圆点（关闭/最小化/最大化）」
- ✅ 「窗口边框跟随用户当前操作系统呈现，原型图仅展示 macOS 风格作示意」

### 3. 品牌/视觉细节不写括号注释

正式 logo 待定时不要在 PRD 写「（紫色方形 logo + 文字）」之类装饰描述。原型用什么视觉是另一回事，PRD 描述的是「有 logo + 有文字」这件事本身。

### 4. 关键交互独立成 bullet

不要把交互塞在状态描述末尾，分行写明确：

- ✅ 独立 bullet：「**「进入」按钮**：符合条件的卡片底部均为圆形「进入」按钮，点击后前往对应详情页（说明 / 操作项等）」

### 5. 失败 / 异常处理独立成 bullet

不要让失败处理塞在「状态流转」末尾。toast 文案、列表保留逻辑、错误后的可恢复路径**单独一条 bullet**：

- ✅「**安装失败处理**：安装失败时弹出 toast 提示「{对象名称} 安装失败，请重试」；任务队列保留该失败任务记录，用户可在任务队列中查看错误详情或重新触发安装」

### 6. 不写「整体视觉」「整体氛围」这类废话 bullet

「浅灰桌面背景 + 圆角窗口 + 投影」属于原型视觉而非产品需求。删。

### 7. 版本号 / 数量等数据展示规则要枚举所有分支

数据格式因状态而异时，逐个状态列出：

- ✅「**版本号展示规则**：已安装的项显示当前已装版本（如 v1.0.45）；**可升级**的项同时展示「当前版本 → 可升级版本」（如 v0.10.0 → v0.11.0）；未安装 / 安装中 / 检测中的项显示「最新版本：vXX」」

## md 与 HTML 面板同步

功能概览内容同时存在于 `.md` 文件和 HTML `prd-panel` `<li>` 中，修改时两处必须同步更新。

HTML 面板结构：

```html
<div class="prd-section">
  <div class="prd-section__title">{编号}：{页面短名}</div>
  <ul>
    <li><strong>元素名</strong>：展示 ...（从对应 md 抄）</li>
  </ul>
</div>
```

每个 section 对应一个 `prd-section`，只装该 section 的 bullets，不混入其他 section。
