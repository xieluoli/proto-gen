# PRD bullets 写法规范

> **适用范围**：设备无关。PC · macOS / Mobile 系列共用同一套 PRD 写法。

每个 section 的 PRD bullets 按**从上到下视觉布局顺序**排列，顶部控件 → 主区内容 → 操作按钮 → 弹窗/抽屉。

## 元素描述模板

| 元素类型 | 描述模板 |
|------|------|
| 静态文本/标题 | `展示「{文案}」{位置/样式}` |
| 内容卡片/列表 | `展示 {元素}，内容来源：{数据源}` |
| 按钮 | `展示「{文案}」按钮（{样式}），点击后{行为}` |
| 链接 | `展示「{文案}」链接，点击后跳转至 {目标}` |
| 状态指示 | `展示 {状态元素}（{文案/颜色}），用于 {含义}` |
| 交互（hover/focus） | `{触发行为}时{反馈}` |
| 条件分支 | `{条件 A}：{元素状态描述}；{条件 B}：{元素另一状态描述}` |

规则：

- 每条单一职责，一个可感知元素一条
- 描述含"是什么"和"什么数据/什么行为"两维
- 数据来源用通俗描述（来自配置 / 来自后端 / 静态 / 由用户输入），不写实现细节
- 冒号使用全角 `：`

### 条件分支型描述

同一控件在不同上下文有不同行为时，用**条件对照**清晰列出每种分支，避免歧义。

示例：

> - 默认 724AI 渠道：「删除」图标渲染为禁用态（灰色 + not-allowed 光标 + 透明 hover 背景），tooltip 文案为「默认渠道不可删除」；点击无任何响应
> - 非默认渠道：「删除」图标使用危险色样式，用于强调破坏性操作

写法要点：

- 每个分支独立一条 bullet（不要在同一行用「如果…则…」并列描述）
- 条件前缀简短（如「默认渠道」「失败态」「未登录用户」），冒号后描述视觉/行为
- 用于同控件不同态、同元素不同用户身份、同字段在不同流程的差异

## 重复内容处理

同一模块内多个 section 共享相同结构时，只在**第一个 section 完整描述**，后续 section 改用引用，降低维护成本。

### 三类常见重复场景

| 场景 | 示例 | 引用写法 |
|------|------|---------|
| **详情页多状态变体**（同模板、不同主按钮/badge） | Home-02 / Home-03 / Home-04（不同安装状态） | `详情页通用结构（返回链接、图标 + 名称 + 副标题、完整简介、操作按钮）同 Home-02` |
| **弹窗通用结构**（遮罩 + X 关闭按钮）在同模块多处弹窗 | Settings-03 / Settings-04（不同弹窗） | `遮罩、X 关闭按钮同 Settings-03；底层为 Settings-01 dimmed 态` |
| **遮罩在同文件连续出现**（抽屉/弹窗叠加） | Settings-04 → Settings-05 | `遮罩同上；[本节独有内容]` |

### 引用规则

1. 第一个引入该结构的 section 必须完整描述，它是后续 section 的锚点
2. 后续 section 先写一行引用，再列差异点（用「差异：」或直接列差异 bullets）
3. 差异内容仍按正常描述模板写，不省略
4. 若两个弹窗内容差异较大，不强制引用，完整描述更清晰

### 正面案例

`model-mgmt.html` 的「模型库-02-编辑」/「模型库-02-编辑-默认」/「模型库-02-删除」三个 section 是引用式差异描述的典范：

- 「模型库-02-编辑」完整描述编辑子页结构
- 「模型库-02-编辑-默认」首行 `结构与「模型库-02-编辑」一致，差异点见下`，之后只列差异（默认渠道额外的 badge、只读字段、恢复按钮等）
- 「模型库-02-删除」首行 `底层背景为「模型库-02-编辑」编辑子页内容...`，复用底层描述，重点写弹窗本身

可参考路径：`724AIManager/MVP/designs/prototype/model-mgmt.html`

## md 与 HTML 面板同步

功能概览内容同时存在于 `.md` 文件和 HTML `prd-panel` `<li>` 中，修改时两处必须同步更新。

HTML 面板结构：

```html
<div class="prd-section">
  <div class="prd-section__title">{编号}：{页面短名}</div>
  <ul>
    <li>展示 ...（从对应 md 抄）</li>
  </ul>
</div>
```

每个 section 对应一个 `prd-section`，只装该 section 的 bullets，不混入其他 section。
